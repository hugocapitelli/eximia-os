Aqui está o passo a passo para criar um PRD:

1. **Input do Usuário**: Recebo a solicitação do usuário para criar um PRD, incluindo informações sobre o problema a ser resolvido, objetivos, e qualquer requisito específico.

2. **Validação do Problema**: Confirmo o problema a ser resolvido. Pergunto: "Qual é a dor específica que estamos solucionando? Para quem?"

3. **Definição do Escopo**: Delimito o que está dentro e fora do escopo do projeto. Pergunto: "O que é essencial e o que não será abordado?"

4. **Estabelecimento de Critérios de Sucesso**: Defino como medir o sucesso do produto. Pergunto: "Quais são as métricas que indicarão que o problema foi resolvido?"

5. **Criação de User Stories**: Escrevo histórias de usuário que descrevem as funcionalidades desejadas. Cada história segue o formato: "Como [persona], eu quero [ação] para que [benefício]."

6. **Elaboração do PRD**: Compilo todas as informações em um documento estruturado, seguindo o template de PRD que inclui:
   - Problem Statement
   - Goals & Non-Goals
   - User Stories
   - Success Metrics
   - Wireframes (se aplicável)
   - Open Questions

7. **Revisão e Validação**: Revisito o PRD com stakeholders para garantir que está claro e atende às necessidades.

8. **Output**: Entrego o PRD finalizado em formato Markdown, pronto para ser utilizado por equipes de desenvolvimento e design.

Esse é o processo direto para a criação de um PRD.
