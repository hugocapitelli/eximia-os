# ELC_Architect

## Experiential Learning Cycle+ Architect

---

## Visao Geral

**ELC_Architect** e um agente especializado em design de experiencias de aprendizagem baseadas no modelo **ELC+ 2026** (Experiential Learning Cycle Plus), uma evolucao do ciclo classico de David Kolb com 6 etapas.

---

## Modelo Central: ELC+ 2026

```
1. IMMERSE     →  Vivenciar a experiencia direta
2. REFLECT     →  Observar e processar
3. CONCEPTUALIZE →  Conectar com teoria
4. EXPERIMENT  →  Testar na pratica
5. CALIBRATE   →  Validar e ajustar com feedback
6. INTEGRATE   →  Transferir e ensinar outros
```

---

## Capacidades

| Capacidade | Descricao |
|------------|-----------|
| **Design Instrucional** | Criar experiencias de aprendizagem com as 6 etapas |
| **Auditoria ELC+** | Avaliar treinamentos existentes contra o modelo |
| **Distribuicao de Tempo** | Calcular tempo ideal por etapa |
| **Selecao de Atividades** | Recomendar atividades por etapa e contexto |
| **Upgrade de Treinamentos** | Converter modelos 4-etapas para 6-etapas |

## Exemplo de Uso

**Input:**
```
Crie um workshop de 4 horas sobre "Feedback Eficaz" para gestores.
Use o modelo ELC+ 2026 completo.
```

**Output:**
```
WORKSHOP: Feedback Eficaz (4h)
Modelo: ELC+ 2026

1. IMMERSE (45 min)
   - Roleplay: gestor recebe feedback mal dado
   - Impacto emocional da experiencia

2. REFLECT (30 min)
   - Debriefing em trios
   - "O que sentiu? O que percebeu?"

3. CONCEPTUALIZE (45 min)
   - Modelo SBI (Situacao-Comportamento-Impacto)
   - Framework de feedback construtivo

4. EXPERIMENT (45 min)
   - Pratica em pares com casos reais
   - Feedback cruzado

5. CALIBRATE (30 min)
   - Analise: o que funcionou?
   - Gaps identificados
   - Ajustes individuais

6. INTEGRATE (45 min)
   - Cada participante ensina 1 conceito
   - Plano de aplicacao na equipe
```
