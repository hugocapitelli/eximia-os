# KB_09: Learning Experience Design Moderno

## Visão Geral

**LXD (Learning Experience Design)** é a evolução do Design Instrucional tradicional, focando na experiência completa do aprendiz, não apenas na instrução.

---

## LXD vs Instructional Design

| Aspecto | ID Tradicional | LXD Moderno |
|---------|----------------|-------------|
| Foco | Conteúdo e objetivos | Experiência do aprendiz |
| Metáfora | Ensinar | Engajar |
| Perspectiva | O que ensinar | Como o aprendiz vivencia |
| Influências | Educação | UX Design + Educação |
| Métricas | Completion, scores | Engagement, aplicação, NPS |

---

## Princípios de LXD (Niels Floor)

### 1. Human-Centered

> *"Design for real people with real needs and challenges."*

- Pesquisa de audiência antes de design
- Empatia com contexto real de trabalho
- Personas de aprendizagem

### 2. Goal-Oriented

- Conectar a resultados de negócio
- Objetivos claros e mensuráveis
- Relevância imediata

### 3. Problem-Based

- Partir de problemas reais
- Contextualização prática
- Aplicação > teoria

### 4. Iterative

- Prototipagem rápida
- Teste com usuários
- Melhoria contínua

### 5. Holistic

- Toda a jornada do aprendiz
- Antes, durante e depois
- Suporte contínuo

## Tendências 2024+

### 1. IA em L&D

| Aplicação | Descrição |
|-----------|-----------|
| Personalização | Trilhas adaptativas por performance |
| Curadoria | IA recomenda conteúdo relevante |
| Avaliação | Análise automatizada de gaps |
| Criação | IA gera rascunhos de materiais |
| Tutoria | AI tutors para prática |

### 2. Skills-Based Learning

- Foco em skills (não cursos)
- Skill graphs e ontologies
- Learning in the flow of work
- Just-in-time microlearning

### 3. Social Learning

- Comunidades de prática
- Peer learning
- User-generated content
- Mentoria em escala

### 4. Imersão e XR

| Tecnologia | Uso |
|------------|-----|
| VR | Treinamento de segurança, soft skills |
| AR | Performance support on-the-job |
| Metaverso | Colaboração e simulações |

### 5. Learning Experience Platforms (LXP)

Evolução do LMS tradicional:
- Curadoria de múltiplas fontes
- Experiência tipo "Netflix"
- Social features
- Skills tracking
- AI recommendations

## Universal Design for Learning (UDL)

### 3 Princípios CAST

| Princípio | Pergunta | Implementação |
|-----------|----------|---------------|
| **Engagement** | Por que aprender? | Múltiplas opções de engajamento |
| **Representation** | O que aprender? | Múltiplos formatos de conteúdo |
| **Action/Expression** | Como demonstrar? | Múltiplas formas de avaliação |

### Guidelines UDL

**Engagement:**
- Oferecer escolhas
- Otimizar relevância
- Minimizar ameaças
- Fomentar auto-regulação

**Representation:**
- Opções de percepção
- Opções de linguagem
- Opções de compreensão

**Action:**
- Opções de ação física
- Opções de expressão
- Opções de função executiva

[CAST, Universal Design for Learning Guidelines, 2018]

## Personalization Types

### Níveis de Personalização

```
Nível 1: SEGMENTAÇÃO
└── Trilhas por cargo/nível
    (Todos do grupo veem o mesmo)

Nível 2: RECOMENDAÇÃO
└── AI sugere com base em comportamento
    (Sugestões personalizadas)

Nível 3: ADAPTAÇÃO
└── Conteúdo muda com base em performance
    (Ajuste dinâmico de dificuldade)

Nível 4: GERAÇÃO
└── Conteúdo criado sob demanda por AI
    (Personalização total)
```

## Learning in the Flow of Work

### Conceito (Josh Bersin)

> *"Bring learning to where work happens, not workers to where learning happens."*

### Implementação

| Momento | Solução |
|---------|---------|
| Dúvida pontual | Microlearning, search |
| Erro/problema | Performance support |
| Feedback | Just-in-time coaching |
| Novo projeto | Curated learning path |
| Curiosidade | Recomendações AI |

## Métricas LXD Modernas

### Além de Completion Rate

| Métrica | O que mede |
|---------|------------|
| **Engagement Score** | Interações, tempo, repeat visits |
| **Application Rate** | % que aplica no trabalho |
| **Learner NPS** | Recomendação |
| **Skill Gain** | Delta de competência |
| **Performance Lift** | Impacto em KPIs |
| **Time to Competency** | Velocidade de desenvolvimento |
