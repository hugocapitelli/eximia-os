# KB_10: Taxonomias de Aprendizagem

## Visão Geral

Taxonomias fornecem linguagem e hierarquia para classificar níveis de complexidade cognitiva em objetivos de aprendizagem.

---

## 1. Taxonomia de Bloom (Revisada)

### Original vs Revisada

| Original (1956) | Revisada (2001) |
|-----------------|-----------------|
| Conhecimento | Lembrar |
| Compreensão | Entender |
| Aplicação | Aplicar |
| Análise | Analisar |
| Síntese | Avaliar |
| Avaliação | Criar |

### Pirâmide Revisada

```
                    ┌─────────┐
                    │ CRIAR   │  ← Mais complexo
                    ├─────────┤
                   /│ AVALIAR │\
                  / └─────────┘ \
                 /  ┌─────────┐  \
                /  /│ ANALISAR│\  \
               /  / └─────────┘ \  \
              /  /  ┌─────────┐  \  \
             /  /  /│ APLICAR │\  \  \
            /  /  / └─────────┘ \  \  \
           /  /  /  ┌─────────┐  \  \  \
          /  /  /  /│ ENTENDER│\  \  \  \
         /  /  /  / └─────────┘ \  \  \  \
        /  /  /  /  ┌─────────┐  \  \  \  \
       /  /  /  /  /│ LEMBRAR │\  \  \  \  \  ← Menos complexo
      └──┴──┴──┴──┴─┴─────────┴─┴──┴──┴──┴──┘
```

### Verbos por Nível

| Nível | Verbos para Objetivos |
|-------|----------------------|
| **Lembrar** | Listar, definir, identificar, nomear, reconhecer |
| **Entender** | Explicar, descrever, classificar, resumir, parafrasear |
| **Aplicar** | Executar, implementar, usar, demonstrar, resolver |
| **Analisar** | Diferenciar, organizar, atribuir, comparar, contrastar |
| **Avaliar** | Criticar, julgar, justificar, validar, priorizar |
| **Criar** | Projetar, construir, desenvolver, produzir, inventar |

[Anderson & Krathwohl, A Taxonomy for Learning, 2001]

## 2. Marzano's New Taxonomy

### 3 Sistemas + Conhecimento

```
┌─────────────────────────────────────────────────────────────────┐
│               MARZANO'S NEW TAXONOMY                             │
│                                                                  │
│   SISTEMA 4: SELF-SYSTEM                                        │
│   └── Motivação, importância, eficácia                          │
│   └── "Por que devo aprender isso?"                             │
│                          ↓                                       │
│   SISTEMA 3: METACOGNITION                                      │
│   └── Monitoramento, planejamento, avaliação do processo        │
│   └── "Como estou aprendendo?"                                  │
│                          ↓                                       │
│   SISTEMA 2: COGNITIVE (Processing)                             │
│   └── Retrieval, Comprehension, Analysis, Utilization           │
│   └── "O que estou fazendo com a informação?"                   │
│                          ↓                                       │
│   SISTEMA 1: KNOWLEDGE                                          │
│   └── Informação, Procedimentos, Psicomotor                     │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### Diferença para Bloom

| Marzano | Bloom |
|---------|-------|
| Inclui Self-System (motivação) | Só cognitivo |
| Metacognição explícita | Implícita |
| Tipos de conhecimento claros | Menos específico |

[Marzano & Kendall, The New Taxonomy of Educational Objectives, 2007]

## 3. Webb's Depth of Knowledge (DOK)

### 4 Níveis de Profundidade

| DOK | Descrição | Verbos |
|-----|-----------|--------|
| **1 - Recall** | Lembrar fatos, procedimentos simples | Definir, identificar, calcular |
| **2 - Skill/Concept** | Aplicar skills, tomar decisões | Classificar, organizar, resumir |
| **3 - Strategic Thinking** | Raciocínio, planejamento, múltiplos passos | Analisar, criar, avaliar |
| **4 - Extended Thinking** | Projetos complexos, tempo estendido | Pesquisar, projetar, sintetizar |

### Matriz DOK + Bloom

```
                    DOK →
                1        2        3        4
              Recall  Skill   Strategic Extended
        ┌──────┬──────┬──────┬──────┬──────┐
Bloom   │Create│      │      │  ◆   │  ◆   │
  ↓     │------│------│------│------│------│
        │Eval  │      │  ◆   │  ◆   │  ◆   │
        │------│------│------│------│------│
        │Analyz│      │  ◆   │  ◆   │  ◆   │
        │------│------│------│------│------│
        │Apply │  ◆   │  ◆   │  ◆   │      │
        │------│------│------│------│------│
        │Undrst│  ◆   │  ◆   │      │      │
```

## 4. SOLO Taxonomy (Biggs)

### Structure of Observed Learning Outcome

| Nível | Descrição | Indicador |
|-------|-----------|-----------|
| **Pre-structural** | Não entende, resposta irrelevante | Sem compreensão |
| **Uni-structural** | Um aspecto correto | Um ponto |
| **Multi-structural** | Vários aspectos, não integrados | Lista de pontos |
| **Relational** | Aspectos integrados, compreensão coerente | Estrutura conectada |
| **Extended Abstract** | Generaliza além do dado, cria novo | Transferência |

### Uso em Assessment

SOLO é útil para:
- Rubrics de avaliação
- Feedback específico de nível
- Diagnóstico de onde o aprendiz está

[Biggs & Collis, Evaluating the Quality of Learning, 1982]

## Comparação de Taxonomias

| Aspecto | Bloom | Marzano | DOK | SOLO |
|---------|-------|---------|-----|------|
| Foco | Complexidade cognitiva | Sistemas mentais | Profundidade | Qualidade de resposta |
| Níveis | 6 | 4 sistemas | 4 | 5 |
| Uso principal | Objetivos | Design instrucional | Rigor de assessment | Feedback/rubric |
| Limitação | Só cognitivo | Complexo | Contexto-dependente | Só outcome |

## Aplicação Prática

### Escrevendo Objetivos de Aprendizagem

**Fórmula ABCD:**
- **A**udience: Quem
- **B**ehavior: O que fará (verbo)
- **C**ondition: Em que situação
- **D**egree: Com que nível de performance

**Exemplo:**
> "Ao final do módulo (C), os gestores (A) serão capazes de conduzir (B) uma reunião de feedback usando o modelo SBI, com 90% de aderência ao checklist (D)."
