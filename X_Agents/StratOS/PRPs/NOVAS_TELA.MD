2. Gestão de Time e Permissões (Admin Settings)
Para atender as personas de Admin, Editor e Viewer e as regras de convite (US-05), falta uma tela de configurações de time. Nela, o Admin visualiza quem tem acesso, altera papéis (RBAC) e monitora convites pendentes que expiram em 7 dias (RN-04).

3. Histórico e Check-in de Metas (KPI Value Entry)
O banco de dados prevê uma tabela de kpi_values com histórico. Falta uma interface onde o Responsável pela Medição (definido no modelo de dados) entra semanalmente ou mensalmente para inserir o valor atual da meta, permitindo que os gráficos de tendência no War Room funcionem com dados reais.

4. Central de Notificações e Atividade
Como o sistema gera alertas proativos da IA, escalações de bloqueios e menções no Catchball, é essencial uma tela (ou gaveta lateral) que consolide todas essas interações para que o usuário não perca prazos ou solicitações de ajuda (US-33 e US-41).

5. Snapshot Estratégico (Public/Stakeholder View)
Para a persona do Observador (Investidor/Board), o PRD solicita um snapshot trimestral (US-24) e link público (FR-6.7). Falta uma tela "Read-Only" otimizada, sem menus de edição, que exiba apenas o progresso consolidado de forma elegante para quem não opera o sistema no dia-a-dia.
