# KB_14: Change Management para Implementação de A3

## Por que Change Management no A3

O A3 define **O QUÊ** fazer (contramedidas). Change Management define **COMO** fazer as pessoas adotarem.

> "A resistência à mudança não é o problema. A resistência é a resposta natural a uma mudança mal gerenciada."

## Modelo de Kotter - 8 Passos

### Aplicação no A3

| Passo Kotter | Aplicação A3 |
|--------------|--------------|
| 1. Criar urgência | Condições Atuais com dados de impacto |
| 2. Formar coalizão | Nemawashi com stakeholders |
| 3. Criar visão | Objetivos e Metas claros |
| 4. Comunicar visão | Compartilhar A3 amplamente |
| 5. Empoderar ação | Remover barreiras no cronograma |
| 6. Gerar quick wins | Priorizar contramedidas de impacto rápido |
| 7. Consolidar ganhos | Monitoramento contínuo |
| 8. Ancorar na cultura | Padronização (Act do PDCA) |

### Erro Comum

Pular diretamente para implementação (Passo 5) sem os passos 1-4:
- Sem urgência → "Por que mudar?"
- Sem coalizão → Resistência isolada
- Sem visão → Direções conflitantes
- Sem comunicação → Mal-entendidos

## ADKAR - Modelo Individual

### Os 5 Elementos

| Elemento | Pergunta | No A3 |
|----------|----------|-------|
| **A**wareness | Por que mudar? | Contexto + Condições Atuais |
| **D**esire | Quero mudar? | Benefícios claros para stakeholders |
| **K**nowledge | Como mudar? | Contramedidas detalhadas |
| **A**bility | Consigo mudar? | Treinamento, recursos, suporte |
| **R**einforcement | Vou continuar? | Monitoramento, reconhecimento |

### Diagnóstico de Resistência

Se a mudança não está acontecendo, identifique ONDE está o bloqueio:

```
Bloqueio em A (Awareness): "Não sabia que tinha problema"
  → Ação: Comunicar dados das Condições Atuais

Bloqueio em D (Desire): "Não vejo benefício para mim"
  → Ação: WIIFM (What's In It For Me)

Bloqueio em K (Knowledge): "Não sei como fazer diferente"
  → Ação: Treinamento, documentação, exemplos

Bloqueio em A (Ability): "Não consigo, falta recurso/tempo"
  → Ação: Remover barreiras, fornecer recursos

Bloqueio em R (Reinforcement): "Fiz uma vez, mas voltei ao antigo"
  → Ação: Incentivos, monitoramento, ajuste de sistema
```

## Análise de Stakeholders

### Mapeamento

Para cada contramedida significativa, mapear:

| Stakeholder | Impacto | Influência | Posição Atual | Posição Desejada |
|-------------|---------|------------|---------------|------------------|
| Nome/Grupo | Alto/Médio/Baixo | Alto/Médio/Baixo | Resistente/Neutro/Apoiador | - |

### Estratégias por Quadrante

```
           ALTA INFLUÊNCIA
                 │
    GERENCIAR    │    ENGAJAR
    DE PERTO     │    ATIVAMENTE
                 │
─────────────────┼─────────────────
                 │
    MONITORAR    │    MANTER
                 │    INFORMADO
                 │
           BAIXA INFLUÊNCIA
     BAIXO IMPACTO    ALTO IMPACTO
```

### Ações por Posição

| Posição | Estratégia |
|---------|------------|
| **Resistente forte** | Entender razões, envolver na solução, não ignorar |
| **Resistente leve** | Comunicação, endereçar preocupações |
| **Neutro** | Informar, mostrar benefícios |
| **Apoiador leve** | Engajar como embaixador |
| **Apoiador forte** | Usar como agente de mudança |

## Comunicação da Mudança

### Framework 7C

1. **Claro**: Linguagem simples, sem jargão
2. **Conciso**: Máximo necessário, mínimo suficiente
3. **Concreto**: Exemplos, dados, não abstrações
4. **Correto**: Informações verificadas
5. **Coerente**: Mensagem consistente em todos os canais
6. **Completo**: Responde as perguntas principais
7. **Cortês**: Respeita o receptor

### Mensagens-Chave por Fase

| Fase A3 | Mensagem | Frequência |
|---------|----------|------------|
| Início | "Por que estamos fazendo isso" (urgência) | Única, impactante |
| Durante | "Como está progredindo" (status) | Semanal/Quinzenal |
| Contramedidas | "O que vai mudar e por quê" (impacto) | Por contramedida |
| Implementação | "O que esperar" (preparação) | Antes de cada ação |
| Pós | "O que conseguimos" (celebração) | Ao atingir marcos |

### Canais por Audiência

| Audiência | Canal Preferido |
|-----------|-----------------|
| Liderança | Reunião executiva, relatório executivo |
| Gerentes | Reunião de gestão, A3 completo |
| Supervisores | Reunião de turno, visual no Gemba |
| Operação | Quadro visual, comunicação verbal direta |
| Parceiros | Email formal, reunião específica |

## Resistência à Mudança

### Causas Comuns

| Causa | Sintoma | Ação |
|-------|---------|------|
| Medo do desconhecido | "Sempre fizemos assim" | Pilotos, demonstrações |
| Perda de status/poder | Boicote silencioso | Envolver na solução |
| Falta de confiança | "Vai funcionar?" | Dados, quick wins |
| Sobrecarga | "Mais uma iniciativa" | Simplificar, priorizar |
| Experiência passada | "Já tentamos isso" | Explicar o que é diferente |

### Técnicas para Lidar

1. **Escuta Ativa**: Entender a resistência antes de responder
2. **Envolvimento**: Incluir resistentes no design da solução
3. **Transparência**: Admitir dificuldades, não prometer milagres
4. **Suporte**: Oferecer ajuda concreta, não apenas cobrança
5. **Paciência**: Mudança leva tempo, não forçar

## Quick Wins (Vitórias Rápidas)

### Por que São Importantes

- Constroem momentum
- Demonstram viabilidade
- Geram credibilidade
- Reduzem resistência

### Critérios para Quick Win

| Critério | Descrição |
|----------|-----------|
| Rápido | Implementável em < 30 dias |
| Visível | Resultado perceptível por stakeholders |
| Mensurável | Melhoria quantificável |
| Relacionado | Conectado ao objetivo maior do A3 |
| Viável | Alta probabilidade de sucesso |

### Identificando no A3

Ao priorizar contramedidas, incluir pelo menos 1 quick win:

```
CONTRAMEDIDAS (priorizadas)

1. [Quick Win] Implementar check visual de temperatura
   - Prazo: 2 semanas
   - Impacto esperado: -20% defeitos de selo

2. [Médio prazo] Automatizar controle de processo
   - Prazo: 3 meses
   - Impacto esperado: -50% defeitos total

3. [Estruturante] Redesenhar layout da linha
   - Prazo: 6 meses
   - Impacto esperado: -30% lead time
```

## Sustentação da Mudança

### Por que Mudanças Falham Após Implementação

1. Falta de monitoramento contínuo
2. Pressão por resultados de curto prazo
3. Turnover de pessoas-chave
4. Falta de atualização do padrão
5. Complacência após sucesso inicial

### Mecanismos de Sustentação

| Mecanismo | Descrição | No A3 |
|-----------|-----------|-------|
| **Padrão documentado** | Novo processo formalizado | Seção Act do PDCA |
| **Treinamento** | Capacitação contínua | Ação no cronograma |
| **Auditoria** | Verificação de conformidade | Parte do monitoramento |
| **Indicador** | Métrica que detecta desvio | Seção Monitoramento |
| **Reconhecimento** | Celebrar quem segue o novo | Ritual de revisão |

### Checklist de Sustentação

- [ ] Padrão atualizado reflete as contramedidas?
- [ ] Todos treinados no novo padrão?
- [ ] Indicador de aderência definido?
- [ ] Auditoria periódica agendada?
- [ ] Dono definido para manutenção?
- [ ] Processo de feedback estabelecido?
