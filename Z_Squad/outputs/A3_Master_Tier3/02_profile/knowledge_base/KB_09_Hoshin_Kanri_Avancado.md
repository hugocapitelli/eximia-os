# KB_09: Hoshin Kanri Avancado

## Origem e Significado

**Hoshin Kanri** (方針管理) significa literalmente:
- **Hoshin** (方針): Direção da bússola, política, plano
- **Kanri** (管理): Gestão, controle

Desenvolvido no Japão nos anos 1960, integrando conceitos de Deming (PDCA) com práticas de gestão japonesas. Bridgestone (1965) e Komatsu foram pioneiros.

## Anatomia do Hoshin Kanri

### Os 7 Passos do Hoshin

1. **Estabelecer Visão Organizacional** (3-5 anos)
2. **Desenvolver Objetivos Breakthrough** (1 ano)
3. **Desdobrar para Departamentos** (Catchball)
4. **Implementar** (A3s Táticos e Operacionais)
5. **Revisar Mensalmente** (Check)
6. **Revisar Anualmente** (Hansei)
7. **Padronizar e Continuar** (Yokoten)

### True North (Norte Verdadeiro)

O **True North** é a visão ideal de longo prazo que guia todas as decisões:

```
TRUE NORTH (Ideal)
     ↑
VISÃO (3-5 anos)
     ↑
OBJETIVOS BREAKTHROUGH (1 ano)
     ↑
INICIATIVAS TÁTICAS (A3 Tático)
     ↑
AÇÕES OPERACIONAIS (A3 Operacional)
     ↑
TRABALHO DIÁRIO (Gemba)
```

## X-Matrix (Matriz X)

### Estrutura

A X-Matrix é a ferramenta visual central do Hoshin Kanri:

```
              OBJETIVOS BREAKTHROUGH (Sul)
                         |
                         |
ESTRATÉGIAS ←————————————+————————————→ MÉTRICAS
(Oeste)                  |              (Leste)
                         |
              TÁTICAS/PROJETOS (Norte)
```

### Os 4 Quadrantes

| Quadrante | Conteúdo | Horizonte |
|-----------|----------|-----------|
| **Sul** | Objetivos Breakthrough | 1-3 anos |
| **Oeste** | Estratégias de Alto Nível | Anual |
| **Norte** | Táticas/Projetos Prioritários | Trimestral/Mensal |
| **Leste** | Métricas e Targets | Contínuo |

### Correlações

As interseções mostram relações:
- ● Forte correlação (responsável primário)
- ○ Correlação moderada (contribuidor)
- △ Correlação fraca (stakeholder)

## Catchball (キャッチボール)

### O Processo de Ida-e-Volta

```
EXECUTIVOS
    ↓ Propõem direção estratégica
    ↑ Recebem feedback de viabilidade
GERENTES
    ↓ Propõem táticas e recursos
    ↑ Recebem feedback de realidade
SUPERVISORES
    ↓ Propõem ações operacionais
    ↑ Recebem feedback do Gemba
OPERADORES
```

### Regras do Catchball

1. **Bidirecional**: Não é top-down puro
2. **Iterativo**: Múltiplas rodadas até alinhamento
3. **Baseado em Fatos**: Dados do Gemba informam viabilidade
4. **Documentado**: A3 é a "bola" que passa entre níveis
5. **Respeitoso**: Divergências são oportunidades de aprendizado

## Bowling Chart

### O que é

Gráfico de acompanhamento mensal que mostra:
- Meta anual (linha de chegada)
- Trajetória planejada (linha ideal)
- Resultado real (linha atual)
- Gap acumulado

### Estrutura

```
Meta ─────────────────────────────────────── 100%
                                    ....../
                              ....../
                        ....../
                  ..../  ← Planejado
            ..../
       ..../
  ──────────────────────────────────────── Real
Jan  Fev  Mar  Abr  Mai  Jun  Jul  Ago  Set  Out  Nov  Dez
```

### Uso no A3

O Bowling Chart deve aparecer na seção de **Monitoramento** do A3 para:
- Visualizar progresso vs meta
- Identificar necessidade de contramedidas adicionais
- Comunicar status para stakeholders

## Hoshin Kanri e A3

### Hierarquia de Documentos

```
HOSHIN KANRI (X-Matrix)
├── A3 ESTRATÉGICO (por Driver)
│   ├── A3 TÁTICO (por Iniciativa)
│   │   ├── A3 OPERACIONAL (por Contramedida)
│   │   └── A3 OPERACIONAL
│   └── A3 TÁTICO
└── A3 ESTRATÉGICO
```

### Mapeamento Hoshin → A3

| Elemento Hoshin | Seção A3 |
|-----------------|----------|
| Objetivo Breakthrough | Contexto (vinculação) |
| Estratégia | Objetivo/Meta do A3 |
| Tática/Projeto | Contramedidas |
| Métrica | Monitoramento |

## Ciclo Anual do Hoshin

### Calendário Típico

| Mês | Atividade |
|-----|-----------|
| Out-Nov | Reflexão (Hansei) do ano atual |
| Nov-Dez | Drafting do Hoshin do próximo ano |
| Dez-Jan | Catchball de alinhamento |
| Jan | Finalização e comunicação |
| Fev-Nov | Execução com revisões mensais |
| Dez | Fechamento e início do novo ciclo |

### Revisão Mensal

Foco da revisão mensal:
1. Status dos indicadores (Bowling Chart)
2. Contramedidas em andamento
3. Novos problemas identificados
4. Decisões necessárias

**Não é**: Apresentação de desculpas ou culpados

## Erros Comuns no Hoshin

| Erro | Consequência | Correção |
|------|--------------|----------|
| Muitos objetivos | Dispersão de foco | Máximo 3-5 breakthrough |
| Catchball superficial | Falta de buy-in | Múltiplas rodadas reais |
| Métricas lagging only | Reação tardia | Incluir leading indicators |
| Revisão como cobrança | Cultura de esconder problemas | Foco em aprendizado |
| Desconexão com Gemba | Hoshin teórico | Validar com operação |

## Integração com Modelo Shingo

O Hoshin Kanri incorpora os princípios Shingo:

- **Pensar Sistemicamente**: X-Matrix mostra conexões
- **Criar Constância de Propósito**: True North mantém direção
- **Criar Valor para Cliente**: Objetivos derivam de necessidades do cliente
- **Liderar com Humildade**: Catchball valoriza input de todos
