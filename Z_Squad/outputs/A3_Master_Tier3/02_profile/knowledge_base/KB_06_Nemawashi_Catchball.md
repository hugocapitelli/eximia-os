# KB_06: Nemawashi e Catchball

## Nemawashi (根回し)

### Significado Literal
"Trabalhar em torno das raízes" - originalmente um termo de jardinagem que se refere ao processo de preparar uma árvore para transplante, cuidando de suas raízes antes de movê-la.

### Aplicação em Negócios
Técnica de construção de consenso que visa remover obstáculos à aprovação de uma proposta, buscando audiência informal com tomadores de decisão ANTES de uma reunião formal.

### Princípio Toyota #13
> "Tome decisões lentamente por consenso, considerando completamente todas as opções; implemente decisões rapidamente (Nemawashi)."

Este estilo de tomada de decisão requer paciência, confiança nos colegas e no processo. A falta de confiança é provavelmente o maior impedimento ao seu uso.

## Por que Nemawashi é Crítico no A3

| Aspecto | Benefício |
|---------|-----------|
| **Decisão lenta** | Mais análise, mais perspectivas, mais qualidade |
| **Implementação rápida** | Obstáculos já removidos, buy-in já conquistado |
| **Reduz conflito** | Issues emergem em conversas 1-a-1, não em reuniões |
| **Preserva harmonia** | Desacordos resolvidos antes da formalização |
| **A3 como ferramenta** | O A3 é a "bola" usada no Catchball |

## Processo de Nemawashi para A3

### Passo 1: Preparar Rascunho
- Elabore versão inicial do A3
- Inclua contexto, problema, análise e propostas

### Passo 2: Identificar Stakeholders
- Quem será impactado?
- Quem tem poder de decisão?
- Quem tem conhecimento relevante?
- Quem pode bloquear?

### Passo 3: Conversas 1-a-1
- Apresente o A3 individualmente
- Peça feedback honesto
- Escute objeções e preocupações
- Documente insights

### Passo 4: Incorporar Feedback
- Revise o A3 baseado nas conversas
- Endereçe objeções diretamente
- Melhore análise com novos dados

### Passo 5: Repetir até Consenso
- Continue iterando até ter alinhamento suficiente
- "Suficiente" = principais stakeholders apoiam

### Passo 6: Apresentação Formal
- Reunião de aprovação é formalidade
- Decisão já foi essencialmente tomada
- Foco na confirmação, não na discussão

## Catchball (キャッチボール)

### O que é
Processo de "jogar a bola" entre níveis hierárquicos no desdobramento estratégico (Hoshin Kanri). É um método de deploy de estratégia que promove comunicação bidirecional e alinhamento em todos os níveis da organização.

### Como Funciona

```
┌─────────────────────────────────────────────────────────────┐
│                    EXECUTIVOS                               │
│           "Jogam a bola" com direção estratégica           │
│                          ↓ ↑                                │
│                    GERENTES                                 │
│     "Pegam" e devolvem com feedback, táticas, recursos     │
│                          ↓ ↑                                │
│                    SUPERVISORES                             │
│     "Pegam" e devolvem com realidade operacional           │
│                          ↓ ↑                                │
│                    COLABORADORES                            │
│     "Pegam" e devolvem com insights do Gemba               │
└─────────────────────────────────────────────────────────────┘
```

### A3 como a "Bola"

Os A3s fornecem foco e disciplina para pessoas refletirem e melhorarem ideias conforme as passam adiante. Assim, os passos de preencher, revisar e monitorar os A3s podem gerar consenso sobre:
- O que estamos fazendo
- Por que estamos fazendo
- Como saberemos se está funcionando

### Benefícios do Catchball

| Benefício | Descrição |
|-----------|-----------|
| **Alinhamento** | Cada pessoa entende seu papel no quadro maior |
| **Ownership** | Ideias moldadas pelas equipes têm mais energia |
| **Reduz fricção** | Issues emergem cedo, antes de derrapar |
| **Engajamento** | Funcionários são contribuidores ativos, não receptores passivos |
| **Qualidade** | Decisões incorporam sabedoria de múltiplos níveis |

### Catchball vs Comunicação Top-Down

| Top-Down Tradicional | Catchball |
|---------------------|-----------|
| Estratégia feita PARA funcionários | Estratégia feita COM funcionários |
| Feedback após implementação | Feedback durante planejamento |
| Resistência na execução | Buy-in na execução |
| Problemas descobertos tarde | Problemas descobertos cedo |
| Ajustes custosos | Ajustes integrados |

## Checklist de Nemawashi para A3

Use antes de qualquer apresentação formal:

- [ ] **Sponsor validou contexto estratégico?**
  - Alinhamento com Hoshin confirmado

- [ ] **Áreas impactadas foram consultadas?**
  - Todos os departamentos afetados participaram

- [ ] **Donos das ações concordam com prazos?**
  - Responsáveis validaram viabilidade

- [ ] **Recursos necessários foram negociados?**
  - Orçamento, pessoas, tempo confirmados

- [ ] **Objeções foram endereçadas?**
  - Preocupações incorporadas ou respondidas

- [ ] **Dados foram validados com Gemba?**
  - Informações verificadas na fonte

## Mitos sobre Nemawashi

### Mito 1: "É só politicagem"
**Realidade**: É construção genuína de consenso baseada em diálogo e fatos.

### Mito 2: "Demora muito"
**Realidade**: Decisão pode ser mais lenta, mas implementação é muito mais rápida.

### Mito 3: "Só funciona no Japão"
**Realidade**: O princípio de buscar alinhamento antes de decisões é universal.

### Mito 4: "Elimina debate"
**Realidade**: Move o debate para conversas 1-a-1 mais produtivas.

## Aplicação Prática

### Em Organizações Ocidentais

1. **Antes de reuniões importantes**: Converse individualmente com participantes-chave
2. **Em projetos**: Colete input de stakeholders antes de finalizar plano
3. **Em A3s**: Circule rascunho e incorpore feedback antes de aprovação
4. **Em mudanças**: Entenda preocupações e endereçe proativamente

### Perguntas para Conversas 1-a-1

- "O que você vê que eu posso estar perdendo?"
- "Quais preocupações você tem com esta abordagem?"
- "O que tornaria isso mais viável para sua área?"
- "Que dados adicionais ajudariam?"
- "Quem mais deveria ser consultado?"
