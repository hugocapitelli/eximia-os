# KB_04: Diagrama de Ishikawa 6M

## Origem

O Diagrama de Ishikawa, também conhecido como Diagrama Espinha de Peixe ou Diagrama de Causa e Efeito, foi popularizado nos anos 1960 por Kaoru Ishikawa, que foi pioneiro em processos de gestão da qualidade nos estaleiros Kawasaki no Japão.

É uma das **7 Ferramentas Básicas da Qualidade** e uma das ferramentas mais amplamente utilizadas na gestão da qualidade.

## O que é

O Diagrama de Ishikawa é uma ferramenta de qualidade que ajuda usuários a identificar as muitas possíveis causas de um problema, organizando ideias em categorias úteis. Ele gera possibilidades, não provas ou causas raiz confirmadas.

## Os 6M da Manufatura

Ishikawa descreve os fatores contribuintes como os 6M no mundo da manufatura:

### 1. MÉTODO (Method)
**Foco**: Processos, procedimentos, padrões operacionais

**Perguntas-Guia**:
- O processo está padronizado?
- O padrão é seguido?
- O padrão é adequado?
- Há etapas que não agregam valor?
- Os procedimentos estão documentados?
- Há ambiguidade nas instruções?

**Causas Típicas**:
- Processo não padronizado
- Padrão desatualizado
- Instruções ambíguas
- Etapas desnecessárias
- Falta de procedimento documentado

### 2. MEDIDA (Measurement)
**Foco**: Indicadores, calibração, coleta de dados

**Perguntas-Guia**:
- Como medimos o problema?
- Os instrumentos estão calibrados?
- A frequência de medição é adequada?
- Os dados são confiáveis?
- O método de coleta é consistente?

**Causas Típicas**:
- Instrumento descalibrado
- Frequência de medição inadequada
- Dados não confiáveis
- Método de coleta inconsistente
- Indicador não reflete realidade

### 3. MÃO DE OBRA (Manpower)
**Foco**: Competências, treinamento, capacidade

**Perguntas-Guia**:
- As pessoas foram treinadas?
- Há matriz de competências?
- A carga de trabalho é adequada?
- Há rotatividade afetando?
- As pessoas têm as ferramentas necessárias?

**Causas Típicas**:
- Falta de treinamento
- Matriz de competência inexistente
- Sobrecarga de trabalho
- Alta rotatividade
- Comunicação deficiente

**IMPORTANTE**: Nunca culpe a pessoa - busque a falha no sistema que permitiu o erro.

### 4. MÁQUINA (Machine)
**Foco**: Equipamentos, sistemas, ferramentas, TI

**Perguntas-Guia**:
- O equipamento é adequado?
- A manutenção está em dia?
- Há capabilidade para o processo?
- Os sistemas suportam a operação?
- O software está atualizado?

**Causas Típicas**:
- Equipamento obsoleto
- Manutenção atrasada
- Máquina não capaz para o processo
- Sistema não integrado
- Ferramenta inadequada

### 5. MATERIAL (Material)
**Foco**: Insumos, informações, matéria-prima

**Perguntas-Guia**:
- A especificação está correta?
- O fornecedor é qualificado?
- O armazenamento é adequado?
- A informação chega no tempo certo?
- O material está dentro da validade?

**Causas Típicas**:
- Especificação incorreta
- Fornecedor não qualificado
- Armazenamento inadequado
- Informação atrasada ou incompleta
- Material fora de especificação

### 6. MEIO AMBIENTE (Mother Nature/Environment)
**Foco**: Condições físicas, cultura organizacional

**Perguntas-Guia**:
- As condições físicas são adequadas?
- Há pressão de prazo excessiva?
- A cultura permite reportar problemas?
- Há comunicação entre áreas?
- O ambiente é seguro?

**Causas Típicas**:
- Temperatura/umidade inadequada
- Pressão excessiva por resultados
- Cultura de culpa
- Silos departamentais
- Ambiente inseguro

## Modelos Expandidos

### 7M (inclui Money)
- **Money**: Recursos financeiros, orçamento, investimentos

### 8M (inclui Management e Maintenance)
- **Management**: Liderança, gestão, tomada de decisão
- **Maintenance**: Manutenção preventiva, preditiva, corretiva

## Critérios de Qualidade do Ishikawa

| Critério | Descrição |
|----------|-----------|
| Quantidade | Mínimo 2, máximo 4 causas por M |
| Verificabilidade | Cada causa deve ser verificável com dados |
| Sistêmico | Causas são sistêmicas, NUNCA pessoais |
| Evidência | Cada causa com evidência ou marcada [VALIDAR] |
| Balanceamento | Cobertura equilibrada entre os 6M |

## Integração com 5 Porquês

O Ishikawa e os 5 Porquês são frequentemente usados juntos:

1. **Ishikawa** gera possibilidades (brainstorming de causas)
2. **Priorização** identifica causas mais prováveis (Pareto, votação, dados)
3. **5 Porquês** aprofunda nas causas prioritárias

## Processo de Construção

### Passo 1: Definir o EFEITO
- Escreva o problema claramente no lado direito (cabeça do peixe)
- Use uma frase única e objetiva

### Passo 2: Desenhar a Estrutura
- Linha horizontal principal (espinha dorsal)
- 6 linhas diagonais (os 6M)

### Passo 3: Brainstorm de Causas
- Para cada M, pergunte: "O que nesta categoria pode estar causando o problema?"
- Anote cada causa como uma linha secundária

### Passo 4: Aprofundar
- Para causas importantes, faça sub-ramificações
- Use perguntas "por quê?" para detalhar

### Passo 5: Priorizar
- Use dados, votação ou Pareto para priorizar
- Foque nas causas com maior impacto provável

### Passo 6: Verificar
- Valide as causas priorizadas com dados do Gemba
- Confirme antes de definir contramedidas

## Quando Usar

### Ideal para:
- Problemas complexos com múltiplas variáveis
- Brainstorming estruturado de causas
- Visão holística do problema
- Equipes multidisciplinares
- Documentação visual de análise

### Complementa:
- Análise de Pareto (priorização)
- 5 Porquês (aprofundamento)
- A3 Thinking (documentação)
- Six Sigma (DMAIC)
