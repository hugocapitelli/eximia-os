# KB_13: Yokoten — Transferência Lateral de Conhecimento

## O que é Yokoten

**Yokoten** (横展) significa literalmente "desdobramento horizontal" ou "expansão lateral".

É a prática sistemática de compartilhar aprendizados de uma área para outras áreas similares, multiplicando os benefícios de uma melhoria.

> "Uma melhoria que fica isolada é uma oportunidade desperdiçada."
> — Princípio Toyota

### Diferença de Benchmark

| Benchmark | Yokoten |
|-----------|---------|
| Copia resultado | Transfere processo |
| Externo | Interno |
| "O que eles fazem?" | "Como podemos adaptar?" |
| Estático | Dinâmico |
| Ocasional | Sistemático |

## Por que Yokoten é Crítico

### O Problema da "Ilha de Excelência"

```
ANTES DO YOKOTEN:
┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐
│ Planta A│  │ Planta B│  │ Planta C│  │ Planta D│
│   ⭐    │  │         │  │         │  │         │
│ Excelente│ │ Média   │  │ Média   │  │  Ruim   │
└─────────┘  └─────────┘  └─────────┘  └─────────┘

DEPOIS DO YOKOTEN:
┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐
│ Planta A│  │ Planta B│  │ Planta C│  │ Planta D│
│   ⭐    │  │   ⭐    │  │   ⭐    │  │   ⭐    │
│ Excelente│ │Excelente│  │Excelente│  │Excelente│
└─────────┘  └─────────┘  └─────────┘  └─────────┘
```

### Benefícios Quantificáveis

- **Velocidade**: Não reinventar a roda
- **Custo**: Menor investimento para mesma melhoria
- **Qualidade**: Aprender com erros dos outros
- **Cultura**: Colaboração vs competição interna

## Tipos de Yokoten

### 1. Yokoten de Processo
Transferir metodologia ou processo bem-sucedido.

**Exemplo**: Setup rápido desenvolvido na linha 1 aplicado nas linhas 2, 3, 4.

### 2. Yokoten de Contramedida
Aplicar contramedida de um problema similar.

**Exemplo**: Poka-yoke criado para evitar erro de montagem no produto A aplicado no produto B.

### 3. Yokoten de A3
Compartilhar A3 bem-sucedido para inspirar outros.

**Exemplo**: A3 que reduziu lead time no armazém São Paulo replicado no Rio de Janeiro.

### 4. Yokoten de Conhecimento
Transferir expertise de pessoas.

**Exemplo**: Especialista em análise de causa raiz do time X treina time Y.

## Processo de Yokoten

### Fluxo de 5 Passos

```
1. IDENTIFICAR     2. DOCUMENTAR      3. ADAPTAR
Melhoria bem   →   O que funcionou?  →  Como aplicar
sucedida          Por quê?              no novo contexto?
    │                   │                    │
    │                   │                    │
    ▼                   ▼                    ▼
                4. IMPLEMENTAR      5. VALIDAR
                Executar com    →   Funcionou?
                ajustes locais      Aprender e refinar
```

### Detalhamento

#### Passo 1: Identificar
- Qual melhoria teve resultado comprovado?
- Onde mais existe problema similar?
- Quem se beneficiaria?

#### Passo 2: Documentar
- O que foi feito (contramedida)?
- Por que funcionou (causa raiz atacada)?
- Quais condições eram necessárias?
- Quais erros evitar?

#### Passo 3: Adaptar
- O contexto novo é similar o suficiente?
- O que precisa ser ajustado?
- Quais condições locais são diferentes?

#### Passo 4: Implementar
- Quem é o owner no novo local?
- Qual suporte do local original?
- Qual cronograma realista?

#### Passo 5: Validar
- Resultado foi alcançado?
- O que funcionou diferente?
- O que aprendemos para próximo Yokoten?

## Yokoten e A3

### Seção de Yokoten no A3

Todo A3 deveria ter uma seção (ou campo) dedicada:

```markdown
## 9. YOKOTEN

### Origem (se aplicável)
- Este A3 foi inspirado em: [A3 original / área origem]
- Adaptações feitas: [o que mudou]

### Potencial de Replicação
- Áreas que podem se beneficiar: [lista]
- Condições necessárias: [pré-requisitos]
- Próximo passo: [ação de Yokoten]

### Registro
- [ ] Melhoria documentada para Yokoten
- [ ] Áreas potenciais notificadas
- [ ] Sessão de compartilhamento agendada
```

### Checklist de Yokoten no A3

Após A3 bem-sucedido:
- [ ] O que aprendemos está documentado?
- [ ] Existe problema similar em outras áreas?
- [ ] Quem deveria saber sobre esta solução?
- [ ] Como vamos compartilhar?
- [ ] Quem é responsável pelo Yokoten?

## Mecanismos de Yokoten

### 1. Sessões de Compartilhamento

**Formato:**
- 30-60 minutos
- Dono do A3 apresenta
- Foco em "por que funcionou", não apenas "o que fizemos"
- Tempo para perguntas e adaptação

**Frequência:**
- Mensal para A3 Táticos
- Trimestral para A3 Estratégicos
- Ad hoc para operacionais de alto impacto

### 2. Biblioteca de A3

**Repositório pesquisável:**
```
A3_Library/
├── Por_Area/
│   ├── Producao/
│   ├── Qualidade/
│   ├── Logistica/
│   └── ...
├── Por_Tipo_Problema/
│   ├── Lead_Time/
│   ├── Defeitos/
│   ├── Seguranca/
│   └── ...
└── Por_Contramedida/
    ├── Poka_Yoke/
    ├── Trabalho_Padronizado/
    ├── Visual_Management/
    └── ...
```

### 3. Rede de Especialistas

**Mapa de competências:**
- Quem resolveu problemas de [tipo]?
- Quem tem expertise em [área]?
- Quem pode fazer coaching em [método]?

### 4. Visitas Cruzadas

**Gemba Walk de Yokoten:**
- Time A visita time B após melhoria
- Observa implementação real
- Discute adaptações para seu contexto

## Barreiras ao Yokoten e Como Superar

| Barreira | Sintoma | Solução |
|----------|---------|---------|
| "Aqui é diferente" | Rejeição sem análise | Focar no "por quê funciona", não no "o quê" |
| Síndrome NIH | "Não inventado aqui" | Reconhecer quem adapta, não só quem cria |
| Falta de tempo | "Estamos muito ocupados" | Integrar Yokoten no processo de A3 |
| Silos | Áreas não conversam | Fóruns cross-funcionais obrigatórios |
| Medo | "Se compartilho, perco vantagem" | Cultura de colaboração, reconhecimento |

## Métricas de Yokoten

| Métrica | Como Medir |
|---------|------------|
| **Taxa de Yokoten** | % de A3 bem-sucedidos que geraram Yokoten |
| **Alcance** | Número de áreas que receberam Yokoten |
| **Velocidade** | Tempo entre A3 original e replicação |
| **Eficácia** | % de Yokotens que alcançaram resultado similar |
| **Multiplicador** | ROI total / ROI original |

**Meta sugerida**: 50% dos A3 bem-sucedidos devem gerar pelo menos 1 Yokoten em 90 dias.
