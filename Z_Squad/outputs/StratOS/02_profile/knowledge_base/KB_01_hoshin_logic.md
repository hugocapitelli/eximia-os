# KB: Lógica do Hoshin Kanri e Desdobramento (Cascade)

## 1. O que é o Hoshin Kanri (nesta metodologia)
É um sistema de gestão estratégica que conecta a Visão de Longo Prazo com a Execução Diária, garantindo alinhamento vertical e horizontal.

Nesta implementação específica (StratOS), **NÃO utilizamos a Matriz X padrão**. Utilizamos uma lógica de fluxo:
`Norte Verdadeiro` -> `Objetivos Estratégicos` -> `Drivers` -> `Indicadores de Resultado` -> `Problemas` -> `Iniciativas`.

## 2. A Cadeia de Causa e Efeito
A estratégia deve ser lida como uma história lógica:
1. **Para atingir** a Visão...
2. **Precisamos entregar** estes Objetivos Estratégicos (What).
3. **Para entregar** os Objetivos, precisamos alavancar estes Drivers (How/Where).
4. **Saberemos se vencemos** medindo estes Indicadores de Resultado.
5. **Hoje não batemos a meta** por causa destes Problemas Raiz.
6. **Para eliminar** os problemas, executaremos estas Iniciativas.

## 3. Regras de Ouro do Desdobramento
- **Objetivos:** Máximo 3 a 5. Devem ser resultados finais (ex: "Aumentar EBITDA"), não projetos (ex: "Implantar SAP").
- **Drivers:** São as áreas de foco ou alavancas. Ex: "Eficiência Operacional", "Expansão de Canais". Um objetivo pode ter 1 ou 2 drivers, não mais.
- **Indicadores:** Devem medir o fim, não o meio.
  - *Errado:* % de treinamento realizado.
  - *Certo:* % de redução de defeitos (resultado do treinamento).
- **Iniciativas:** Só existem para fechar o gap do indicador. Se o indicador já está na meta, não precisa de iniciativa.

## 4. O Papel do Catchball
O Catchball é a negociação entre níveis.
- A liderança define "O Quê" (Objetivos).
- A operação define "Como" (Iniciativas).
- O acordo é firmado sobre os "Recursos" e "Prazos".
**StratOS deve simular esse diálogo**, perguntando: "Com esses recursos, você confia que esse indicador chegará na meta?"
