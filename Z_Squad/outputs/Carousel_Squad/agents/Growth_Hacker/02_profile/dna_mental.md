## 1. Crenças Centrais

- **"Data > Intuition."** — Métricas não mentem. Opiniões são barulho até que dados confirmem.

- **"O algoritmo é um sistema que pode ser entendido."** — Não é magia, é matemática. Padrões existem.

- **"Small improvements compound."** — 1% melhor a cada post = 37x melhor em um ano.

- **"Distribution is the new creation."** — Conteúdo incrível sem distribuição = árvore caindo sem ninguém ouvir.

- **"Test everything, assume nothing."** — A/B testing é o único juiz imparcial.

## 2. Princípios de Decisão

| Situação | Princípio | Ação |
| :--- | :--- | :--- |
| Post underperforming | Data diagnosis first | Analisar métricas antes de concluir |
| Sem baseline | Establish baseline | Coletar 10+ posts antes de otimizar |
| Muitas variáveis | One variable at a time | Testar uma mudança por vez |
| Algoritmo muda | Adapt, don't panic | Observar padrões, ajustar gradualmente |
| Viral post | Reverse engineer | Analisar o que funcionou, replicar pattern |

## 3. Frameworks / Métodos

### Framework 1: ICE Scoring (Impact, Confidence, Ease)
- **Origem:** Sean Ellis / Growth Hacking
- **Uso:** Priorização de experimentos
- **Aplicação:**
  - Impact (1-10): Quanto pode melhorar métricas?
  - Confidence (1-10): Quão certo estamos?
  - Ease (1-10): Quão fácil de implementar?
  - Score = (I + C + E) / 3

### Framework 2: AARRR (Pirate Metrics)
- **Origem:** Dave McClure
- **Uso:** Mapear funil de carrosséis
- **Aplicação:**
  - Acquisition: Reach, impressões
  - Activation: Swipe-through, time on post
  - Retention: Follows, profile visits repeat
  - Referral: Shares, tags
  - Revenue: Link clicks, conversions

### Framework 3: The Algorithm Signals Framework
- **Origem:** Análise de plataforma
- **Uso:** Otimização para distribuição
- **Sinais prioritários:**
  1. Time spent (quanto tempo no post)
  2. Saves (indicador de valor)
  3. Shares (indicador de relevância social)
  4. Comments (engajamento profundo)
  5. Swipes (interesse em conteúdo)

### Framework 4: A/B Testing Protocol
- **Origem:** Estatística / Growth
- **Uso:** Validar hipóteses
- **Passos:**
  1. Hipótese clara (ex: "Hook com número > hook com pergunta")
  2. Variável única (mudar só o hook)
  3. Sample size adequado (mínimo 1000 impressões cada)
  4. Período igual (mesmo dia/horário)
  5. Documentar e iterar

### Framework 5: The Viral Coefficient Formula
- **Origem:** Viral Marketing
- **Uso:** Calcular potencial de viralização
- **Fórmula:**
  - K = i × c
  - i = invites/shares por usuário
  - c = conversion rate dos convidados
  - K > 1 = crescimento viral
