## Filosofia de Experimentacao

### O Mindset de Growth Hacker

```
"Growth hacking e um processo, nao uma tatica.
O processo e testar, medir, aprender, iterar."
— Sean Ellis

"Cada post e um experimento. Cada metrica e um aprendizado."
— Adaptado de Neil Patel
```

### Principios Fundamentais

```
1. HIPOTESE ANTES DE ACAO
   └── Nunca teste sem saber o que espera aprender

2. UMA VARIAVEL POR VEZ
   └── Multiplas mudancas = impossivel saber o que funcionou

3. AMOSTRA SUFICIENTE
   └── Conclusoes precipitadas = decisoes erradas

4. DOCUMENTACAO RELIGIOSA
   └── Experimento nao documentado = conhecimento perdido

5. FAIL FAST, LEARN FASTER
   └── Experimento "fracassado" = dado valioso
```

## Categorias de Experimentos

### 1. HOOK Experiments

```
VARIAVEIS TESTAVEIS:
├── Estilo de headline (pergunta vs afirmacao)
├── Presenca de numeros
├── Emocao principal (curiosidade vs urgencia vs medo)
├── Tamanho do texto
├── Posicionamento do texto
└── Uso de emojis

METRICAS PRINCIPAIS:
├── Swipe-Through Rate (slide 1→2)
├── Stop Rate (impressions/reach)
└── Time on first slide
```

### 2. TIMING Experiments

```
VARIAVEIS TESTAVEIS:
├── Horario do dia
├── Dia da semana
├── Timing relativo (antes vs depois de eventos)
└── Frequencia de postagem

METRICAS PRINCIPAIS:
├── Reach
├── Engagement nas primeiras 2 horas
└── Engagement total
```

### 3. VISUAL Experiments

```
VARIAVEIS TESTAVEIS:
├── Paleta de cores
├── Tipografia
├── Estilo de imagens (foto vs ilustracao)
├── Densidade de texto
├── Layout/composicao
└── Branding elements

METRICAS PRINCIPAIS:
├── Engagement Rate
├── Save Rate
└── Profile visits
```

### 4. COPY Experiments

```
VARIAVEIS TESTAVEIS:
├── Tamanho da caption
├── Estrutura (lista vs paragrafo)
├── Tom de voz
├── Uso de emojis
├── Call-to-action
└── First line (hook da caption)

METRICAS PRINCIPAIS:
├── Comments
├── Saves
└── Shares
```

### 5. FORMAT Experiments

```
VARIAVEIS TESTAVEIS:
├── Numero de slides
├── Progressao de conteudo
├── Tipo de conteudo (educativo vs entretenimento)
├── Carrossel vs Reel vs Image
└── Estrutura narrativa

METRICAS PRINCIPAIS:
├── Completion Rate
├── Engagement Rate
└── Reach
```

### 6. HASHTAG Experiments

```
VARIAVEIS TESTAVEIS:
├── Quantidade de hashtags
├── Mix de tamanhos
├── Posicao (caption vs comentario)
├── Especificidade
└── Rotacao de sets

METRICAS PRINCIPAIS:
├── Reach from hashtags
├── Non-follower reach %
└── Hashtag impressions
```

## Priorizacao de Experimentos

### ICE Scoring

```
IMPACT (1-10):
├── 10: Potencial de dobrar metrica-chave
├── 7-9: Melhoria significativa esperada
├── 4-6: Melhoria moderada esperada
├── 1-3: Melhoria marginal esperada

CONFIDENCE (1-10):
├── 10: Dados fortes suportam hipotese
├── 7-9: Evidencias razoaveis
├── 4-6: Intuicao + algumas evidencias
├── 1-3: Puro palpite

EASE (1-10):
├── 10: Implementacao trivial
├── 7-9: Facil, pouco esforco
├── 4-6: Esforco moderado
├── 1-3: Muito trabalhoso

ICE SCORE = (I + C + E) / 3
```

## Cadencia de Experimentacao

### Ritmo Recomendado

```
INICIANTE (aprendendo o sistema):
├── 1 experimento por mes
├── 4 posts por variacao
└── Analise aprofundada

INTERMEDIARIO (sistema rodando):
├── 2 experimentos por mes
├── 4-6 posts por variacao
└── Testes paralelos em categorias diferentes

AVANCADO (otimizacao continua):
├── 4+ experimentos por mes
├── Testes always-on em multiplas categorias
└── Framework automatizado de analise
```

### Sprint de Experimentacao (Mensal)

```
SEMANA 1: Planejamento
├── Review de experimentos anteriores
├── Selecao de novos experimentos (ICE)
├── Setup de configuracao

SEMANA 2-3: Execucao
├── Rodar experimentos
├── Coletar dados
├── Observacoes diarias

SEMANA 4: Analise
├── Consolidar resultados
├── Documentar aprendizados
├── Planejar proximo ciclo
```

## Playbook de Aprendizados

### Estrutura do Playbook

```
PLAYBOOK DE [CATEGORIA]

## REGRAS VALIDADAS
(Experimentos com resultados conclusivos)

1. [REGRA]: [Explicacao]
   - Fonte: EXP-XXX
   - Impacto: +XX% em [metrica]
   - Implementado desde: [data]

2. [REGRA]: [Explicacao]
   ...

## HIPOTESES EM TESTE
(Ainda nao conclusivas)

1. [HIPOTESE]: [Status atual]
2. ...

## MITOS DERRUBADOS
(O que achavamos que funcionava, mas nao funciona)

1. [MITO]: [O que descobrimos]
   - Fonte: EXP-XXX
2. ...
```

### Exemplo de Entrada no Playbook

```
## REGRA VALIDADA: Hook com Numero Especifico

**Regra:** Headlines com numeros impares e especificos (ex: "7" vs "algumas")
aumentam swipe-through rate.

**Fonte:** EXP-HOOK-003, EXP-HOOK-007

**Dados:**
- "7 erros" vs "Varios erros": +23% STR
- "5 passos" vs "Passos para": +18% STR

**Impacto:** +20% medio em Swipe-Through Rate

**Implementado desde:** Marco 2025

**Excecoes:** Conteudo muito emocional pode funcionar sem numeros
```

## Erros Comuns

### 1. Nao Documentar

**Problema:** "Lembro que testei isso, mas nao sei o resultado"
**Solucao:** SEMPRE documentar, mesmo experimentos "obvios"

### 2. Testar Muito Cedo

**Problema:** Concluir com 2 posts de cada
**Solucao:** Minimo 4-6 posts por variacao para reduzir variancia

### 3. Multiplas Variaveis

**Problema:** Mudou cor, texto E horario no mesmo teste
**Solucao:** UMA variavel por experimento

### 4. Ignorar Contexto

**Problema:** Comparar post de feriado com dia normal
**Solucao:** Controlar para variaveis externas

### 5. Confirmation Bias

**Problema:** Parar teste quando resultado confirma o que queria
**Solucao:** Definir duracao ANTES e respeitar
