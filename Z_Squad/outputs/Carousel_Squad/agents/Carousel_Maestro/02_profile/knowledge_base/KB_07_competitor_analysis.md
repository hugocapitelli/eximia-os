# KB_07 — Competitor Analysis Framework

**Agente:** Carousel Maestro
**Categoria:** ANÁLISE COMPETITIVA
**Versão:** 1.0

---

## Por que Analisar Concorrentes?

A análise de concorrentes em carrosséis NÃO é sobre copiar — é sobre:

1. **Identificar lacunas** — O que ninguém está fazendo?
2. **Validar formatos** — O que já funciona no nicho?
3. **Acelerar aprendizado** — Economizar meses de testes
4. **Diferenciar** — Encontrar seu ângulo único
5. **Benchmark** — Saber se suas métricas são boas

---

## Framework SCOPE de Análise Competitiva

### S - Style (Estilo Visual)
### C - Content (Conteúdo)
### O - Outcomes (Resultados)
### P - Patterns (Padrões)
### E - Engagement (Interação)

---

## Passo 1: Identificar Concorrentes Relevantes

### Tipos de Concorrentes

| Tipo | Descrição | O que analisar |
|------|-----------|----------------|
| **Diretos** | Mesmo nicho, mesmo público | Tudo |
| **Indiretos** | Público similar, oferta diferente | Formatos e hooks |
| **Aspiracionais** | Onde você quer chegar | Estratégias avançadas |
| **De formato** | Excelentes em carrossel (qualquer nicho) | Design e estrutura |

### Lista de Verificação para Seleção

```
□ Publica carrosséis regularmente (mínimo 2x/semana)
□ Tem engajamento consistente
□ Público-alvo similar ao seu
□ Está crescendo (não estagnado)
□ Conteúdo de qualidade reconhecida
```

### Template de Mapeamento


---

## Erros Comuns a Evitar

1. **Copiar literalmente** — Adapte, não copie
2. **Analisar demais, criar de menos** — 80% criação, 20% análise
3. **Ignorar o que funciona** — Se todos fazem, tem motivo
4. **Seguir quem não cresce** — Analise quem tem resultados
5. **Não documentar** — Insights se perdem sem registro
