# KB_13 — Quality Assurance Checklist

**Agente:** Carousel Maestro
**Categoria:** CONTROLE DE QUALIDADE
**Versão:** 1.0

---

## O Sistema de QA do Carousel Maestro

Todo carrossel passa por um processo rigoroso de Quality Assurance antes da publicação. Este sistema garante:

1. **Consistência** — Padrão de qualidade em todos os posts
2. **Profissionalismo** — Sem erros que comprometam credibilidade
3. **Performance** — Otimizado para engajamento
4. **Compliance** — Alinhado com brand guidelines

---

## Checklist por Tipo de Carrossel

### QA Específico: Educational/How-To

```
□ Os passos estão em ordem lógica?
□ Cada passo é executável?
□ Não falta nenhum passo crítico?
□ Resultado final está claro?
□ Nível de dificuldade está adequado à audiência?
```

### QA Específico: Listicle

```
□ Todos os itens entregam valor?
□ Não há repetição entre itens?
□ Melhor item está no início ou fim?
□ Número de itens corresponde ao prometido no hook?
□ Itens são comparáveis em qualidade?
```

### QA Específico: Storytelling

```
□ Arco narrativo está completo?
□ Há tensão/conflito?
□ Resolução é satisfatória?
□ Lição está clara (sem ser preachy)?
□ Humaniza sem expor demais?
```

### QA Específico: Data/Statistics

```
□ Todas as fontes são citadas?
□ Dados são recentes (< 12 meses)?
□ Visualização dos dados é clara?
□ Não há interpretação enganosa?
□ Conclusões são suportadas pelos dados?
```

### QA Específico: Promotional/BOFU

```
□ Benefícios estão claros?
□ Prova social está presente?
□ Objeções foram endereçadas?
□ CTA é específico e urgente?
□ Não promete demais (compliance)?
```

## Processo de Revisão

### Etapa 1: Auto-Review

O criador do carrossel executa primeira revisão usando checklist master.

```
TEMPO: 5-10 minutos
QUEM: Próprio criador
FOCO: Erros óbvios, alinhamento com brief
```

### Etapa 2: Peer Review

Segundo par de olhos revisa o carrossel.

```
TEMPO: 10-15 minutos
QUEM: Colega ou outro agente do squad
FOCO: Perspectiva externa, clareza, impacto
```

### Etapa 3: Final Approval

Carousel Maestro dá aprovação final.

```
TEMPO: 5 minutos
QUEM: Carousel Maestro
FOCO: Alinhamento estratégico, qualidade geral
```

## Níveis de Severidade de Problemas

| Nível | Descrição | Ação | Exemplos |
|-------|-----------|------|----------|
| **CRÍTICO** | Impede publicação | Corrigir imediatamente | Erro factual, dado errado, link quebrado |
| **ALTO** | Compromete qualidade | Corrigir antes de publicar | Erro de ortografia visível, design inconsistente |
| **MÉDIO** | Melhoria desejável | Corrigir se possível | Tom levemente off, caption poderia ser melhor |
| **BAIXO** | Nice-to-have | Anotar para próximo | Otimizações menores |

### Critérios de Bloqueio (No-Go)

```
NUNCA PUBLICAR SE:

✗ Há erro factual ou dado incorreto
✗ Há erro de ortografia/gramática grave
✗ Texto está ilegível no mobile
✗ Promete algo que não pode entregar
✗ Viola guidelines de marca
✗ Copia outro criador sem crédito
✗ Expõe informação confidencial
✗ Viola policies da plataforma
```

## Checklist de Última Hora (2 min antes de publicar)

```
FINAL CHECK (60 segundos)

□ Slides estão na ordem correta?
□ Texto principal está sem erros?
□ Visual está OK no preview mobile?
□ Caption está completa?
□ Hashtags estão presentes?
□ Horário de publicação está correto?

SE TUDO ✓ → PUBLICAR
SE QUALQUER ✗ → PAUSAR E CORRIGIR
```

## Métricas de QA

### KPIs do Processo de QA

| Métrica | Meta | Como Medir |
|---------|------|------------|
| Taxa de aprovação 1a rodada | >80% | Aprovados / Total |
| Tempo médio de revisão | <20 min | Tempo por carrossel |
| Issues críticos encontrados | 0 pós-publicação | Erros reportados |
| Retrabalho | <10% | Carrosséis refeitos |
