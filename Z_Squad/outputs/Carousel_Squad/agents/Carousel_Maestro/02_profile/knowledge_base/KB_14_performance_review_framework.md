**Agente:** Carousel Maestro
**Categoria:** ANÁLISE DE PERFORMANCE
**Versão:** 1.0

---

## Por que Revisar Performance?

A análise sistemática de performance permite:

1. **Identificar padrões** — O que funciona repetidamente?
2. **Otimizar recursos** — Investir no que dá resultado
3. **Iterar rapidamente** — Melhorar com base em dados
4. **Evitar erros** — Não repetir o que falhou
5. **Provar ROI** — Demonstrar valor do trabalho

## Métricas Fundamentais a Rastrear

### Métricas Primárias

| Métrica | Fórmula | O que indica |
|---------|---------|--------------|
| **Reach** | Contas únicas alcançadas | Tamanho do impacto |
| **Impressions** | Vezes que foi visto | Frequência de exposição |
| **Engagement Rate** | (Likes+Comments+Saves+Shares)/Reach × 100 | Ressonância geral |
| **Save Rate** | Saves/Reach × 100 | Valor percebido |
| **Share Rate** | Shares/Reach × 100 | Potencial viral |
| **Comment Rate** | Comments/Reach × 100 | Engajamento profundo |

### Métricas Secundárias

| Métrica | O que indica |
|---------|--------------|
| **Swipe-Through Rate** | Interesse no conteúdo |
| **Completion Rate** | Retenção até o final |
| **Profile Visits** | Interesse na marca |
| **Follows** | Conversão de audiência |
| **Link Clicks** | Intenção de ação |

## Framework de Análise de Comentários

### Categorização de Comentários

| Categoria | O que indica | Ação |
|-----------|--------------|------|
| **Perguntas** | Interesse, dúvidas | Criar conteúdo follow-up |
| **Elogios** | Ressonância | Identificar o que foi elogiado |
| **Tags** | Shareability | Manter elementos que geram tags |
| **Histórias** | Identificação | Tema conectou emocionalmente |
| **Objeções** | Gaps | Endereçar em próximos conteúdos |
| **Spam** | N/A | Ignorar no cálculo |


## Framework de Decisão: O Que Fazer com os Dados

### Árvore de Decisão

```
CARROSSEL PERFORMOU BEM?
│
├── SIM (acima da média)
│   │
│   ├── Por que funcionou?
│   │   └── Documentar elementos de sucesso
│   │
│   └── AÇÕES:
│       □ Criar variação do tema
│       □ Replicar estrutura em outro pilar
│       □ Testar hook em outro formato
│
└── NÃO (abaixo da média)
    │
    ├── O que falhou?
    │   └── Identificar causa raiz
    │
    └── AÇÕES:
        □ Testar abordagem diferente
        □ Abandonar tema se repetido
        □ Ajustar timing/formato
```
