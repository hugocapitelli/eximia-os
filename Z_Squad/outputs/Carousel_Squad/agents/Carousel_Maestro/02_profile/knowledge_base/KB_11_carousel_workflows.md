## O Sistema de Produção de Carrosséis

Um workflow eficiente transforma ideias em carrosséis publicados de forma consistente e escalável. Este KB documenta os processos passo a passo para cada etapa.

## Fase 1: Ideação

### Workflow 1.1: Brainstorm de Tópicos

**Duração:** 30 minutos/semana

**Processo:**

```
PASSO 1: COLETA DE INPUTS (10 min)
□ Revisar perguntas recorrentes do público
□ Verificar trends da semana
□ Analisar top posts de concorrentes
□ Consultar banco de ideias existente

PASSO 2: BRAIN DUMP (10 min)
□ Timer de 10 minutos
□ Listar TODAS as ideias sem filtro
□ Mínimo 20 ideias por sessão

PASSO 3: CATEGORIZAÇÃO (5 min)
□ Agrupar por pilar de conteúdo
□ Classificar por tipo de carrossel
□ Marcar estágio do funil (TOFU/MOFU/BOFU)

PASSO 4: PRIORIZAÇÃO (5 min)
□ Aplicar matriz impacto x esforço
□ Selecionar top 5 para próxima semana
□ Adicionar restante ao banco de ideias
```

### Workflow 1.2: Validação de Ideia

**Checklist de validação:**

```
A ideia passa nos critérios?

□ RELEVÂNCIA: Minha audiência se importa com isso?
□ EXPERTISE: Tenho conhecimento para falar disso?
□ DIFERENCIAL: Tenho um ângulo único?
□ TIMING: É o momento certo para esse conteúdo?
□ FORMATO: Funciona como carrossel?
□ OBJETIVO: Qual o resultado esperado?

Se 5+ ✓ → Aprovar para briefing
Se 3-4 ✓ → Revisar e ajustar
Se <3 ✓ → Descartar ou guardar
```

## Fase 2: Briefing

### Workflow 2.1: Pesquisa de Conteúdo

**Duração:** 20-60 minutos por carrossel

**Processo:**

```
PASSO 1: DEFINIR ESCOPO (5 min)
• O que este carrossel vai ensinar/mostrar?
• Qual a transformação prometida?
• Qual o nível de profundidade?

PASSO 2: COLETAR INFORMAÇÕES (15-45 min)
□ Fontes primárias (experiência própria)
□ Fontes secundárias (artigos, livros)
□ Dados e estatísticas relevantes
□ Exemplos e casos práticos

PASSO 3: ORGANIZAR (10 min)
□ Agrupar por subtópico
□ Identificar os pontos principais
□ Selecionar os melhores exemplos
□ Verificar fontes (se necessário citar)
```

## Fase 3: Criação

### Workflow 3.1: Copywriting

**Duração:** 30-60 minutos por carrossel

**Processo por slide:**

```
SLIDE 1 - HOOK
□ Escrever 3 opções de hook
□ Testar contra checklist de hook
□ Selecionar o mais forte
□ Ajustar para formato visual

SLIDES 2-N - DESENVOLVIMENTO
Para cada slide:
□ Um ponto principal por slide
□ Texto conciso (máx. 50 palavras)
□ Formatação visual (bullets, números)
□ Transição para próximo slide

SLIDE FINAL - CTA
□ Resumo opcional
□ CTA claro e específico
□ Alinhado com objetivo do carrossel
```

**Checklist de copy:**

```
□ Cada slide tem 1 ideia principal?
□ O texto cabe no slide visualmente?
□ Há progressão lógica entre slides?
□ O tom está consistente com brand voice?
□ O CTA é claro e acionável?
```

### Workflow 3.2: Design

**Duração:** 45-90 minutos por carrossel

**Processo:**

```
PASSO 1: SETUP (5 min)
□ Abrir template de marca
□ Configurar dimensões corretas
□ Preparar paleta de cores

PASSO 2: SLIDE 1 - CAPA (15 min)
□ Aplicar texto do hook
□ Escolher/criar visual de apoio
□ Garantir hierarquia clara
□ Verificar legibilidade

PASSO 3: SLIDES INTERNOS (20-40 min)
□ Manter consistência visual
□ Aplicar texto de cada slide
□ Adicionar elementos gráficos
□ Verificar espaçamento

PASSO 4: SLIDE FINAL (10 min)
□ Design do CTA
□ Logo/marca se aplicável
□ Elemento de encerramento

PASSO 5: REVISÃO (10 min)
□ Visualizar em sequência
□ Checar alinhamentos
□ Verificar ortografia
□ Testar no celular
```

### Workflow 3.3: Revisão

**Checklist de QA antes de aprovar:**

```
CONTEÚDO
□ Informações estão corretas?
□ Ortografia e gramática OK?
□ Tom consistente com marca?
□ Valor entregue é claro?

DESIGN
□ Visual está branded?
□ Texto legível no celular?
□ Imagens em alta qualidade?
□ Consistência entre slides?

TÉCNICO
□ Dimensões corretas?
□ Formato de arquivo adequado?
□ Ordem dos slides correta?
□ Pronto para publicação?
```

## Fase 4: Publicação

### Workflow 4.1: Caption

**Template de caption:**

```
[HOOK - primeira linha que aparece]
_______________________________________________

[DESENVOLVIMENTO - valor adicional, contexto]
_______________________________________________
_______________________________________________

[CTA - ação específica]
_______________________________________________

[HASHTAGS - 3-5 relevantes]
#___ #___ #___ #___ #___
```

### Workflow 4.2: Agendamento

**Processo:**

```
□ Carregar slides na plataforma
□ Adicionar caption
□ Configurar data/horário
□ Verificar preview
□ Agendar publicação
□ Adicionar lembrete de engajamento
```

### Workflow 4.3: Pós-Publicação

**Checklist de engajamento (primeiras 2 horas):**

```
□ Responder todos os comentários
□ Reagir com likes
□ Compartilhar em stories
□ Engajar com posts relacionados
□ Responder DMs recebidas
```

## Workflows por Tipo de Carrossel

### Workflow: Educational How-To

```
1. Definir resultado final (o que a pessoa saberá fazer)
2. Mapear os passos necessários (5-8 passos)
3. Para cada passo:
   - Título do passo
   - Ação específica
   - Dica ou exemplo
4. Criar hook com promessa clara
5. CTA para salvar/aplicar
```

### Workflow: Listicle

```
1. Definir tema e número (7 dicas, 10 ferramentas...)
2. Listar todos os itens possíveis
3. Selecionar os melhores (priorizar por valor)
4. Ordenar estrategicamente (melhor no início e fim)
5. Um slide por item
6. Hook com número + promessa
7. CTA para salvar
```

### Workflow: Storytelling

```
1. Identificar o arco narrativo
   - Situação inicial
   - Conflito/desafio
   - Turning point
   - Resolução
   - Moral/lição
2. Distribuir em slides (seguindo tensão narrativa)
3. Hook emocional (gancho na história)
4. Slide final com lição + CTA
```

### Workflow: Data/Stats

```
1. Identificar dado principal (mais impactante)
2. Coletar dados de suporte
3. Verificar fontes e atualidade
4. Visualizar cada dado (gráfico, número grande, etc.)
5. Hook com dado mais chocante
6. Conclusão com insight acionável
7. Citar fontes no final ou caption
```

## Troubleshooting

| Problema | Causa Provável | Solução |
|----------|---------------|---------|
| Demora na ideação | Sem sistema | Implementar brain dump semanal |
| Copy fraca | Falta de pesquisa | Aumentar tempo de briefing |
| Design inconsistente | Sem template | Criar template master |
| Atrasos frequentes | Pipeline mal definido | Implementar lead time de 7 dias |
| Baixa qualidade | Sem revisão | Adicionar etapa de QA |
