# KB_05 — Decision Principles

## 1. Principles (Ray Dalio)

### Core Framework (Dalio, Principles, 2017)
```
Pain + Reflection = Progress
```

### Decision-Making Principles

| Principle | Application | CFO Context |
| :--- | :--- | :--- |
| **Radical Transparency** | Share data openly | Financial dashboards |
| **Believability-weighted** | Expertise matters | Consult specialists |
| **5 Whys** | Root cause | Variance analysis |
| **Expected Value** | Probability × Outcome | Investment decisions |

### Pain-to-Benefit Ratio
Segundo Dalio (Principles, 2017):
```
If Pain-to-Benefit Ratio > 2-3x, proceed with caution
```

## 2. Margin of Safety (Graham/Buffett)

### Framework (Graham, The Intelligent Investor, 1949)
```
Fair Value = Intrinsic Value × (1 - Safety Margin)
```

### Safety Margins por Tipo de Ativo

| Asset Type | Recommended Margin | Source |
| :--- | :--- | :--- |
| Mature, stable | 15-20% | Graham (1949) |
| Growth, volatile | 30-40% | Buffett Letters |
| Distressed | 50%+ | Graham (1949) |
| M&A targets | 20-30% | McKinsey (2020) |

### Buffett's Rules (Berkshire Letters, 1987)
1. Rule #1: Never lose money
2. Rule #2: Never forget Rule #1
3. Corollary: Always have margin of safety

## 3. Management by Objectives (Drucker)

### Framework (Drucker, The Effective Executive, 1967)

**5 Práticas do Executivo Eficaz:**
1. Manage time — "Know where your time goes"
2. Focus on contribution — "What can I contribute?"
3. Build on strengths — Não tente eliminar fraquezas
4. Focus on few priorities — "First things first"
5. Effective decisions — Processo estruturado

### Aplicação em Finance
| Prática | Implementação CFO |
| :--- | :--- |
| Time management | CFO time = decisões >R$100k |
| Contribution focus | "What creates enterprise value?" |
| Strength building | Delegar contabilidade operacional |
| Priority focus | Top 3 KPIs, não 20 |
| Decision process | DCF + Scenarios + Review |

## 4. Probabilistic Thinking (Nate Silver)

### Framework (Silver, The Signal and the Noise, 2012)

**Signal vs Noise:**
- Signal: Padrões verdadeiros nos dados
- Noise: Variação aleatória
- Erro comum: Confundir noise com signal

### Bayesian Updating
```
P(H|E) = P(E|H) × P(H) / P(E)

Posterior = (Likelihood × Prior) / Evidence
```

### Aplicação Prática
| Situação | Base Rate (Prior) | Update com Evidência |
| :--- | :--- | :--- |
| M&A success | 30-50% | Adjust with DD findings |
| Startup Series A | 10% makes it to exit | Adjust with metrics |
| Synergy realization | 60-70% | Adjust with integration progress |

## 5. Cognitive Biases (Kahneman)

### Framework (Kahneman, Thinking Fast and Slow, 2011)

| Bias | Definition | CFO Mitigation |
| :--- | :--- | :--- |
| **Overconfidence** | Too certain in projections | Use ranges, not points |
| **Anchoring** | First number sticks | Challenge initial valuation |
| **Confirmation** | Seek confirming evidence | Devil's advocate review |
| **Sunk Cost** | Throw good money after bad | Incremental analysis only |
| **Planning Fallacy** | Underestimate time/cost | Add 20-50% buffer |

### Pre-Mortem Technique
Antes de decisão (Klein, 1989):
"Imagine we are 1 year in the future and this failed. Why did it fail?"

## 6. Decision Framework Integrado

### CFO Decision Checklist

```markdown
□ 1. Define: O que exatamente estou decidindo?
□ 2. Data: Tenho os dados necessários? (Se <60%, parar e coletar)
□ 3. Frameworks: Quais frameworks aplicar? (DCF, Comps, etc.)
□ 4. Scenarios: Bear/Base/Bull analysis
□ 5. Biases: Que vieses podem me afetar?
□ 6. Safety: Qual é a margem de segurança?
□ 7. Expected Value: Probability × Outcome
□ 8. Review: Quem é belieable para revisar?
□ 9. Decision: GO / NO-GO / HOLD
□ 10. Post-mortem: Schedule para validar
```

## Invariantes de Decision-Making

1. **Pain-to-Benefit > 2-3x: HALT** (Dalio)
2. **Margin of Safety: ≥20%** (Graham)
3. **Decisão > R$100k: ≥48h** (Drucker time mgmt)
4. **Uncertainty > 60%: Use ranges** (Kahneman)
5. **Update priors with evidence** (Silver/Bayes)
