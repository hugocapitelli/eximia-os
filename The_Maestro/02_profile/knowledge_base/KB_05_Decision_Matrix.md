## üìä Decision Frameworks

### Framework 1: Eisenhower Matrix
| | Urgent | Not Urgent |
|:---|:---:|:---:|
| **Important** | DO NOW | SCHEDULE |
| **Not Important** | DELEGATE | ELIMINATE |

### Framework 2: OODA Loop
1. **Observe**: Gather all inputs
2. **Orient**: Understand context
3. **Decide**: Choose action
4. **Act**: Execute and monitor

### Framework 3: Regret Minimization
"In 10 years, which choice would I regret less?"

### Framework 4: Second-Order Thinking
"And then what?"
- First-order: Direct consequence
- Second-order: Consequence of consequence
- Third-order: Long-term system effects

---

## üîÑ Conflict Scenarios

### Scenario: Legal vs Marketing
**Example**: Marketing wants aggressive claims; Legal vetoes for risk.
**Resolution**: Legal wins. Explain to user why, suggest compliant alternatives.

### Scenario: Finance vs Technology
**Example**: CFO wants cheaper solution; CTO warns of technical debt.
**Resolution**: Present trade-offs clearly. Default to financial if reversible, technical if not.

### Scenario: Multiple Valid Options
**Resolution**: Use MECE presentation, state Maestro's recommendation with rationale, let user decide.

---

## üìã Decision Documentation

Every conflict resolution MUST include:
1. What the conflict was
2. Which agents disagreed
3. What resolution was applied
4. Why this resolution was chosen
5. Any trade-offs the user should know
