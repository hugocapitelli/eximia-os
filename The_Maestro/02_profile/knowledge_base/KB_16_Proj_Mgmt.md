# KB_16: Project Management Methods

> **Category**: STRATEGY  
> **Purpose**: Frameworks for managing complex multi-step work

---

## ðŸŽ¯ When to Apply

Apply project management when:
- Request requires multiple agents
- Work spans multiple turns
- Dependencies exist between tasks

---

## ðŸ“‹ Methodologies

### Agile (Default)
- Break work into small increments
- Deliver value early and often
- Adapt based on feedback

### Waterfall
- Sequential phases
- Clear handoffs
- Use for regulatory/compliance work

### Kanban
- Visualize work
- Limit work in progress
- Continuous flow

---

## ðŸ”„ Project Structures

### PDCA Cycle
1. **Plan**: Define scope and approach
2. **Do**: Execute the work
3. **Check**: Validate results
4. **Act**: Adjust based on findings

### Sprint Template
```
Sprint Goal: [Specific outcome]
Duration: [This conversation/session]
Tasks:
1. [ ] Task A
2. [ ] Task B
3. [ ] Task C
Definition of Done: [Criteria]
```

---

## ðŸ“Š Critical Path

For complex requests:
1. Map all tasks
2. Identify dependencies
3. Find longest path (critical path)
4. Optimize bottlenecks
5. Parallelize where possible
