**Palavras:** ~2.000

## 1. Tipos de Crise Corporativa

| Tipo | Exemplos | Impacto |
|------|----------|---------|
| **Operacional** | Acidente, contaminação, recall | Financeiro + Reputacional |
| **Legal** | Investigação, condenação | Financeiro + Legal |
| **Reputacional** | Escândalo, viral negativo | Brand equity |
| **Financeira** | Insolvência, fraude | Sobrevivência |
| **Cibernética** | Data breach, ransomware | Multi + LGPD |

---

## 2. War Room

### 2.1 Composição

```
WAR ROOM (CRISIS COMMITTEE)
├── CLO/General Counsel (Legal)
├── CEO/COO (Decisão)
├── CFO (Financeiro)
├── CISO/CTO (TI/Segurança)
├── Comunicação/PR
├── RH (se envolver pessoas)
└── Advogado externo (se necessário)
```

### 2.2 Protocolo de Ativação

| Fase | Ação | Timeline |
|------|------|----------|
| **Detecção** | Identificar e escalar | 0-1h |
| **Avaliação** | Severity assessment | 1-4h |
| **Contenção** | Medidas imediatas | 4-24h |
| **Comunicação** | Stakeholders internos/externos | 24-72h |
| **Remediação** | Ações corretivas | 72h+ |
| **Aprendizado** | Post-mortem | 30 dias |

## 3. Data Breach Response

### 3.1 Protocolo LGPD

```
1. DETECÇÃO (0h)
   - Identificar escopo
   - Isolar sistemas afetados
   
2. AVALIAÇÃO (0-24h)
   - Natureza dos dados
   - Número de titulares
   - Risco de danos
   
3. COMUNICAÇÃO ANPD (24-72h)
   - Se risco relevante
   - Formulário padrão ANPD
   
4. COMUNICAÇÃO TITULARES (72h+)
   - Se alto risco
   - Medidas de proteção
   
5. DOCUMENTAÇÃO
   - Registro de incidente
   - Evidências de resposta
   - Lições aprendidas
```

### 3.2 Não Pagar Resgate

| Argumento | Descrição |
|-----------|-----------|
| Legal | FCPA/UK Bribery (se agente público) |
| Prático | Não garante recuperação |
| Ético | Financia crime organizado |
| Reputacional | Sinaliza vulnerabilidade |

**Recomendação CLO:** NÃO pagar resgate. Acionar seguro cyber, investigar, remediar.

## 4. Investigação Interna

### 4.1 Quando Conduzir

- Denúncia via canal
- Suspeita de fraude
- Requisição regulatória
- Preparação para leniência

### 4.2 Protocolo

| Fase | Ação |
|------|------|
| **Legal Hold** | Preservar documentos |
| **Escopo** | Definir perímetro |
| **Equipe** | Interna ou externa |
| **Entrevistas** | Protocolo padronizado |
| **Análise** | Documentos, dados, e-mails |
| **Relatório** | Conclusões e recomendações |

### 4.3 Privilege Protection

| Medida | Descrição |
|--------|-----------|
| Advogado present | Em entrevistas |
| Marcação | "Privileged & Confidential" |
| Separação | Fato vs opinião legal |
| Controle | Acesso limitado |

## 5. Comunicação de Crise

### 5.1 Princípios

| Princípio | Descrição |
|-----------|-----------|
| **Velocidade** | Primeiro a comunicar |
| **Transparência** | Fatos conhecidos |
| **Empatia** | Mostrar preocupação |
| **Ação** | O que está sendo feito |
| **Consistência** | Uma mensagem, todos canais |

### 5.2 Stakeholder Map

| Stakeholder | Canal | Timing |
|-------------|-------|--------|
| Colaboradores | E-mail, intranet | Primeiro |
| Clientes | E-mail, comunicado | 24h |
| Investidores | Fato relevante (CVM) | Imediato se aplicável |
| Imprensa | Press release | Após stakeholders internos |
| Reguladores | Comunicação formal | Conforme obrigação legal |

---

## 6. Fato Relevante (CVM)

### 6.1 Quando Divulgar

Fato que possa:
- Influir de modo significativo
- Na cotação dos valores mobiliários
- Ou na decisão dos investidores

### 6.2 Conteúdo

| Elemento | Descrição |
|----------|-----------|
| Descrição do fato | O que ocorreu |
| Impacto | Efeitos esperados |
| Providências | Ações adotadas |
| Contato | IR/DRI |

**Timeline:** Imediato, antes da próxima abertura do pregão.

---

## 7. Aplicação CLO

### 7.1 Checklist de Crise

- [ ] War Room ativado
- [ ] Legal hold implementado
- [ ] Advogado externo acionado (se necessário)
- [ ] Comunicação interna preparada
- [ ] Reguladores notificados (se obrigatório)
- [ ] Seguro acionado
- [ ] Documentação de decisões

### 7.2 Papel do CLO na Crise

| Função | Descrição |
|--------|-----------|
| Legal advisor | Orientação jurídica |
| Risk assessor | Quantificar exposição |
| Privilege guardian | Proteger comunicações |
| Regulatory liaison | Interface com reguladores |
| Document controller | Preservação de evidências |
