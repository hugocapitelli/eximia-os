## 1. First Principles Thinking

### O que é
Método de raciocínio que decompõe problemas até verdades fundamentais irredutíveis, rejeitando analogias e convenções.

### Quando Usar
- Problema parece "impossível" ou "caro demais"
- Indústria está estagnada
- Soluções existentes são insatisfatórias
- Pressupostos parecem arbitrários

### Processo (5 Passos)

```
1. IDENTIFICAR o problema ou custo
   "Foguetes custam $65 milhões"

2. LISTAR pressupostos
   - Fornecedores cobram preços fixos
   - Fabricação requer precisão militar
   - Etc.

3. QUESTIONAR cada pressuposto
   "Por que isso é verdade? É lei física ou convenção?"

4. DECOMPOR até elementos fundamentais
   "Quais são os materiais brutos? Alumínio, carbono, titânio..."
   "Quanto custam esses materiais? ~2% do preço final"

5. RECONSTRUIR solução
   "Se materiais custam 2%, posso construir por 10-20% do convencional"
```

### Exemplo Aplicado
**Problema:** "Bateria de carro elétrico é cara demais"
**Decomposição:**
- Custo células: $X/kWh
- Custo materiais brutos (lítio, níquel, cobalto): $Y/kWh
- Gap X-Y = oportunidade de otimização
**Solução:** Integrar célula na estrutura, mudar química, escalar produção

## 2. Rapid Iteration / Fail Fast

### O que é
Preferir protótipos imperfeitos a planos perfeitos. Aprender através de falhas reais, não simulações.

### Quando Usar
- Incerteza alta
- Mercado novo
- Tecnologia nova
- Planos não capturam complexidade

### Processo

```
1. PROTOTIPAR versão mínima
2. TESTAR no mundo real
3. FALHAR (esperado!)
4. ANALISAR falha
5. ITERAR rapidamente
6. REPETIR
```

### Mantra
> "Sucesso é tropeçar de falha em falha sem perder entusiasmo."

### Exemplo
SpaceX Falcon 1:
- Voo 1: Falha (incêndio motor)
- Voo 2: Falha (oscilação propelente)
- Voo 3: Falha (estágio colidiu)
- Voo 4: Sucesso!

Cada falha = aprendizado específico → correção → próxima tentativa

## 3. Vertical Integration

### O que é
Controlar toda a cadeia de produção quando mercado não oferece componentes adequados.

### Quando Usar
- Fornecedores não entregam qualidade necessária
- Custos de mercado são inflados
- Velocidade de inovação é crítica
- Componente é estratégico

### Processo

```
1. TENTAR usar fornecedor
2. Se INADEQUADO → considerar fazer in-house
3. CALCULAR: Custo desenvolvimento vs dependência
4. Se estratégico → INTERNALIZAR
```

### Exemplo Tesla
- Baterias: Parceria Panasonic → Desenvolvimento próprio (4680)
- Chips: Dependia de Mobileye → Chip FSD próprio
- Superchargers: Infraestrutura própria

## 4. Physics as Arbiter

### O que é
Usar leis da física como teste definitivo de possibilidade.

### Quando Usar
- Alguém diz "impossível"
- Convenções parecem arbitrárias
- Débito sobre viabilidade

### Processo

```
1. PERGUNTAR: "É fisicamente impossível?"
   - Viola termodinâmica?
   - Viola velocidade da luz?
   - Viola conservação de energia?

2. Se NÃO → então é questão de engenharia
   - Engenharia é resolvível
   - Só precisa de tempo e recursos

3. Se SIM → buscar alternativa fundamental
```

### Exemplo
"Foguetes reutilizáveis são impossíveis"
- Física permite? Sim (pouso é mecanicamente possível)
- Então é engenharia → resolver

## 5. Manufacturing as Moat

### O que é
A vantagem competitiva real está no PROCESSO de fabricação, não só no produto.

### Quando Usar
- Competidores podem copiar design
- Escala é importante
- Custos de produção determinam viabilidade

### Frase-chave
> "Protótipo é a parte fácil. A máquina que faz a máquina é o hard part."

### Exemplo
Tesla Gigafactory:
- Não é só fábrica, é vantagem competitiva
- Competidores podem copiar Model 3, não podem copiar eficiência de produção

## 6. Impossible Timeline Theory

### O que é
Definir prazos agressivos (aparentemente impossíveis) para forçar inovação e evitar complacência.

### Quando Usar
- Projeto tende a arrastar
- Equipe está otimizando em vez de inovar
- Urgência é real mas não sentida

### Processo

```
1. DEFINIR prazo que parece impossível
2. ACEITAR que vai atrasar (mas não dizer isso)
3. FORÇAR soluções criativas para tentar cumprir
4. Mesmo atrasando, termina MAIS CEDO que com prazo "realista"
```

### Trade-off Reconhecido
- Prazos frequentemente não cumpridos
- Mas: resultado final chega antes do que chegaria com prazo conservador
